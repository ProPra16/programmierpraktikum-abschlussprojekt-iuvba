Nutzerhandbuch

Beim öffnen sollen zwei Dateien ausgewählt werden. Eine Datei beinhaltet den Quellcode für das Programm, die andere Datei beinhaltet dern Quellcode für die Tests.

GREEN
Wenn beide Dateien geladen wurden, öffnet sich das Fenster Green. Dort werden die Tests implementiert. Es muss ein fehlschalgender Test implementiert sein oder der Code darf nicht kompilieren, bevor man in den Red Teil gehen kann.
Wenn man den fehlschlagenden Test implementiert hat, wird durch klicken auf den Check-Button getestet, ob ein Test fehl schlägt oder der Code nicht kompiliert.
Wenn dies der Fall ist, wird ein Weiter-Button sichtbar, mit dem man zum Red Teil weitergeleitet wird.

RED
Im Red Teil wird der passende Code implementiert.
Es gibt dort wieder einen Check-Button, der den Code auf Kompilierbarkeit und fehlschlagende Tests testet.
Bei erfolgreichem Check kann man durch den Weiter-Button zum Refactor Teil wechseln.

REFACTOR
In diesem Teil sollte man den Code optimieren.
Durch klicken auf den Weiter-Button, wird der Code auf Kompilierbarkeit und fehlschlagende Tests getestet.
Bei erfolgreichem Check gelangt man durch klicken auf den Weiter-Button wieder zum Green Teil gelangen
