Nutzerhandbuch

Ziel des Programms ist es, die testgetriebene Programmierung zu erlernen. Dafür ist das Programm in Teile aufgeteilt:
RED:      Schreiben des Tests
GREEN:    Schreiben der Codes
REFACTOR: Code optimieren
Wie es genau funktioniert wird unten beschrieben.

Beim öffnen soll eine JSON Datei ausgewählt werden.
Nach dem Klicken auf den "Weiter" Button erscheint ein Fenster mit zwei Arbeitsplätzen, jeweils ein Platz für RED und GREEN.
Zuerst ist das Bearbeiten nur bei RED (linke Seite) möglich.
 

RED
Dort werden die Tests implementiert. Es muss ein fehlschalgender Test implementiert sein oder der Code darf nicht kompilieren, bevor man in den GREEN Teil gehen kann.
Wenn man den fehlschlagenden Test implementiert hat, wird durch klicken auf den Check-Button getestet, ob ein Test fehl schlägt oder der Code nicht kompiliert.
Wenn dies der Fall ist, wird ein Weiter-Button sichtbar, mit dem man zum GREEN Teil weitergeleitet wird.

GREEN
Hier ist das Bearbeiten im GREEN Feld möglich. Im  diesem Teil wird der passende Code implementiert.
Es gibt dort  einen Check-Button, der den Code auf Kompilierbarkeit und fehlschlagende Tests testet.
Bei erfolgreichem Check kann man durch den Weiter-Button zum Refactor Teil wechseln.


REFACTOR
In diesem Teil sollte man den Code optimieren.
Durch klicken auf den Weiter-Button, wird der Code auf Kompilierbarkeit und fehlschlagende Tests getestet.
Bei erfolgreichem Check gelangt man durch klicken auf den Weiter-Button wieder zum Green Teil gelangen
